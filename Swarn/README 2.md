# Caerus — 4-node Swarm (Terraform + Ansible)

| VM            | Swarm role | Services                                  |
|---------------|-----------|--------------------------------------------|
| caerus-edge   | manager   | nginx (load balancer), redis, Prometheus, Grafana |
| caerus-db     | worker    | MySQL                                      |
| caerus-app-1  | worker    | php-fpm                                    |
| caerus-app-2  | worker    | php-fpm                                    |

Roles are assigned as **node labels** (`caerus_role=edge|db|app`) by Ansible, and
each service is pinned with a **placement constraint** in `caerus-stack.yml`.
nginx forwards to `app:9000`; Docker's service VIP load-balances across the two
app replicas automatically (your `default.conf` already re-resolves it via the
embedded DNS resolver, so no nginx change is needed).

## Layout
```
caerus-swarm/
├── caerus-stack.yml
├── terraform/   (main.tf, variables.tf, outputs.tf, inventory.tftpl)
└── ansible/     (playbook.yml ; inventory.ini is generated by Terraform)
```

## 0) Build & push the images (once)
Swarm does not build. Docker Hub is the simplest registry (nodes pull over NAT):
```bash
docker build -t <your-user>/caerus-app:1.0 -f docker/app/Dockerfile .
docker build -t <your-user>/caerus-web:1.0 -f docker/nginx/Dockerfile .
docker push <your-user>/caerus-app:1.0
docker push <your-user>/caerus-web:1.0
```
Then set those names in `caerus-stack.yml` (replace `registry.example.com/...`).
Fully offline alternative: run an insecure local registry on the edge and add it
to `insecure-registries` on every node (ask and I'll add the Ansible tasks).

## 1) Create the VMs
```bash
cd terraform
terraform init
terraform apply        # check box_image + hostonly_interface first
```
This also writes `ansible/inventory.ini` with the four nodes already grouped.

## 2) Configure the cluster and deploy
```bash
cd ../ansible
ansible-galaxy collection install community.docker
ansible-playbook -i inventory.ini playbook.yml -e caerus_root=/ABS/PATH/TO/Caerus
```

## 3) Open it (edge IP from `terraform output`)
- App:        `http://<edge_ip>:8000`
- Grafana:    `http://<edge_ip>:3000`  (admin / admin)
- Prometheus: `http://<edge_ip>:9090`

## Demo / scaling
```bash
# ssh vagrant@<edge_ip>
docker node ls                      # 4 nodes, roles visible
docker stack services caerus
docker service ps caerus_app        # one task on app-1, one on app-2
docker service scale caerus_app=2   # bounded by max_replicas_per_node=1 (2 app VMs)
```
To scale app beyond 2, add more app VMs (extend `node_names`) and re-apply.

## Notes / next steps
- **Passwords**: MySQL uses Docker secrets; the app gets `DB_PASSWORD` as plain
  env (your app image doesn't read `*_FILE`). Same value on both sides.
- **Prometheus with 2 replicas**: the current scrape hits `web:80/metrics`, so it
  samples whichever app replica the VIP routes to. For per-replica metrics it
  should discover tasks via `tasks.app` — happy to wire that next.
- `depends_on` is ignored by Swarm; ordering relies on healthchecks + restarts.
