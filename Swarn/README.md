# Caerus Swarm — Multipass + Terraform (Apple Silicon)

Provisions 4 native arm64 Ubuntu VMs with Multipass via Terraform, then configures
the Swarm with the same Ansible playbook as before.

| VM            | role  | services                                   |
|---------------|-------|--------------------------------------------|
| caerus-edge   | edge  | nginx (LB), redis, Prometheus, Grafana     |
| caerus-db     | db    | MySQL                                      |
| caerus-app-1  | app   | php-fpm                                    |
| caerus-app-2  | app   | php-fpm                                    |

Only the provisioning layer changed (Multipass instead of VirtualBox). The
`playbook.yml` and `caerus-stack.yml` are reused as-is.

## Prerequisites (one time)
```bash
brew install --cask multipass
# an SSH key for Ansible (skip if you already have ~/.ssh/id_ed25519):
ssh-keygen -t ed25519
```

## 0) Build & push your images (arch matches now)
You build on Apple Silicon (arm64) and the VMs are arm64, so a normal build works.
Push to Docker Hub (the VMs pull over their NAT internet):
```bash
docker build -t <user>/caerus-app:1.0 -f docker/app/Dockerfile .
docker build -t <user>/caerus-web:1.0 -f docker/nginx/Dockerfile .
docker push <user>/caerus-app:1.0
docker push <user>/caerus-web:1.0
```
Set those names on the two `image:` lines in `caerus-stack.yml`.

## 1) Create the VMs (Terraform)
```bash
cd terraform-multipass
terraform init
terraform apply        # builds 4 VMs, writes ../ansible/inventory.ini
terraform output       # shows the edge IP you'll open in the browser
```

## 2) Configure + deploy (Ansible)
```bash
cd ../ansible
ansible-galaxy collection install community.docker
ansible-playbook -i inventory.ini playbook.yml -e caerus_root=/ABS/PATH/TO/Caerus
```

## 3) Open it (IP from `terraform output edge_ip`)
- App:        `http://<edge_ip>:8000`
- Grafana:    `http://<edge_ip>:3000`
- Prometheus: `http://<edge_ip>:9090`

## One required edit to caerus-stack.yml
Remove the `healthcheck:` block under the **app** service. The `php:8.2-fpm`
image has no `nc`, so that check would always fail and Swarm would keep
restarting the app. (Everything else stays the same.)

## Useful Multipass commands
```bash
multipass list                  # names + IPs
multipass shell caerus-edge     # get a shell on a node
multipass stop --all            # pause the lab
multipass delete --all --purge  # tear it down
```

## Notes
- IPs are DHCP (192.168.64.x) and may differ each rebuild — that's why the
  inventory is generated by Terraform rather than hard-coded.
- VMs share the Multipass network and can reach each other, so the Swarm overlay
  (ports 2377/7946/4789) forms with no extra firewall work.
- The `todoroff/multipass` provider is community-maintained; if `terraform apply`
  ever can't read an IP, run `multipass list` to confirm the VM booted, then
  `terraform apply` again.
